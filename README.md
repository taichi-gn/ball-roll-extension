# 🎱 Ball Roll - Chrome拡張機能

ページのボタンやリンクの上を、物理演算でボールが転がるChrome拡張機能。

## 何ができる？

- ページ内のボタン・リンク・入力欄・見出しが**物理的な床**になる
- ボールが上から落ちてきて、それらの上を**転がる・跳ねる**
- ボール同士もぶつかる（物理演算）
- スクロールやリサイズしてもDOM位置を自動で再計算

---

## セットアップ手順

### 1. フォルダの場所を確認

```
~/Desktop/ball-roll-extension/
```

以下のファイルが入っていればOK：
- `manifest.json`
- `content.js`
- `popup.html`
- `popup.js`
- `matter.min.js`

### 2. Chromeで拡張機能ページを開く

アドレスバーに入力：

```
chrome://extensions/
```

### 3. デベロッパーモードをON

画面右上の **「デベロッパーモード」** のトグルを**ON**にする。

### 4. 拡張機能を読み込む

左上の **「パッケージ化されていない拡張機能を読み込む」** をクリック。

フォルダ選択ダイアログで `~/Desktop/ball-roll-extension` を選ぶ。

### 5. ピン留め（任意）

ツールバー右端のパズルピースアイコン🧩をクリック → 🎱 Ball Roll の📌ピンアイコンをクリックしてピン留め。

これでツールバーに🎱アイコンが常駐する。

## 使い方

### 基本操作

1. 任意のWebページを開く（例: Twitter/X, GitHub, Notion）
2. ツールバーの **🎱 アイコン** をクリック
3. ポップアップの **「ボールを落とす」** ボタンをクリック

### ポップアップのボタン

| ボタン | 動作 |
|---|---|
| **ボールを落とす** | 画面上部から1個落下 |
| **10個追加** | 一気に10個落下（パーティーモード） |
| **リセット** | 全ボール削除 |

### プレイヤー干渉操作

- **ボールをドラッグ**: 掴んで投げる（素早く振ると勢いよく飛ぶ）
- **Shift + クリック**: その位置に新しいボール生成
- **Fキー押しっぱなし**: カーソル周辺のボールを吹き飛ばす（Fan/扇風機モード）
- **スクロール**: 障害物判定を自動で再計算
- **リサイズ**: ウィンドウサイズ変えてもOK

### 障害物として判定される要素

- `<button>`
- `<a>` (リンク)
- `<input>`, `<textarea>`, `<select>`
- `[role="button"]`
- `<h1>`, `<h2>`, `<h3>` (見出し)
- `.card` クラス
- `[data-ball-target]` 属性を持つ要素

薄い青の枠線で囲まれた部分がボールが乗る「床」。

## トラブルシューティング

### ボールが表示されない

1. `chrome://extensions/` で拡張機能が **有効** になっているか確認
2. ページリロード（⌘+R）してから再度クリック
3. 拡張機能の詳細 → 「ビューを検査：ポップアップ」でConsoleを確認
4. ページ側のConsole（F12）に `🎱 Ball Roll injected` と出ているか確認

### 一部のサイトで動かない

- Chrome Web Storeページ、`chrome://` 系の内部ページでは動かない（Chromeの制約）
- 厳しいCSP（Content Security Policy）のサイトではブロックされる場合あり
  - GitHub、Slack等はCSP厳しめ
  - 別のサイト（ニュースサイト等）で試してみる

### コード修正後、反映されない

1. `chrome://extensions/` を開く
2. 拡張機能カードの🔄（更新）アイコンをクリック
3. 対象ページをリロード

### アイコンが表示されない

`manifest.json` に `icons` 指定が無いため、デフォルトアイコン（パズルピース）が表示される。アイコン付けたい場合は `icon.png`（16×16, 48×48, 128×128）を用意して `manifest.json` に追加：

```json
"icons": {
  "16": "icon16.png",
  "48": "icon48.png",
  "128": "icon128.png"
}
```

## 技術スタック

- **Manifest V3**（Chromeの最新拡張機能仕様）
- **matter.js** 0.20.0（2D物理演算ライブラリ）
- **Canvas API**（ボールの描画）
- **バニラJS**（フレームワーク無し）

### ファイル構成

```
ball-roll-extension/
├── manifest.json    # 拡張機能の設定
├── popup.html       # ツールバークリック時のUI
├── popup.js         # popup → content.jsへコマンド送信
├── content.js       # 物理演算＋描画のメインロジック
├── matter.min.js    # 物理演算ライブラリ
└── README.md        # このファイル
```

### 仕組み

1. ポップアップボタンを押すと `chrome.scripting.executeScript` で `content.js` と `matter.min.js` を現在のタブに注入
2. `content.js` が DOM を走査し、ボタン等の位置を `getBoundingClientRect()` で取得
3. 取得した矩形を matter.js の静的ボディとして登録（= 物理的な床）
4. 画面全体に透過Canvasを被せてボールを描画
5. `pointer-events: none` でクリック透過 → ページ操作は邪魔しない

## カスタマイズ例

### ボールの重力を変える

`content.js` の以下を編集：

```js
engine.gravity.y = 1.0;  // → 0.3 にするとゆっくり落下
```

### ボールの数を増やす

`popup.js` の `addMany` の `10` を変更。

### 障害物として扱う要素を追加

`content.js` の `selector` を編集：

```js
const selector = 'button, a, input, textarea, select, [role="button"], h1, h2, h3, .card, [data-ball-target]';
```

例: `p` タグも追加したいなら末尾に `, p` を足す。

### 障害物の枠線を非表示に

`content.js` の `render()` 内、`ctx.strokeStyle = 'rgba(59, 130, 246, 0.15)'` の処理をコメントアウト。
